Class: TreeView

TreeView

This is a simple helper class to set up the tree-view data source in the context of a hypergrid.

It includes methods to:

  • Insert DataSourceTreeview into the data model's pipeline (addPipe, addPipeTo).
  • Perform the self-join and rebuild the index to turn the tree-view on or off, optionally hiding the ID columns (setRelation).

Methods

addPipe(data, sharedopt)

Add the tree-view data source into the instance pipeline.

The tree-view data source is inserted into the pipeline of the given data model instance, immediately after the raw data source.

If necessary, a private copy of the prototype's pipeline array is cloned for use by the instance (unless shared is truthy).

Finally, setData is called again with data to rebuild the pipeline. To avoid this, consider TreeView#addPipeTo.

Parameters:
Name Type Attributes Default Description
data Array.<object>

Required for the setData call.
shared boolean <optional>
 false

Do not clone prototype's pipeline array. The default is to clone it.

addPipeTo(dataModelPrototype)

Add the tree-view data source into the shared pipeline.

The tree-view data source is inserted into the shared pipeline of the given data model's prototype, immediately after the raw data source.

The resulting pipeline addition is shared by all new grids using this data model.

Intended to be called on the TreeView prototype, before the data model is instanced (which currently happens when the behavior is instanced (which currently happens when the grid is instanced)).
